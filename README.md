# 🚀 AI 기반 심플 할 일 관리 앱 (Simple Todo App with AI)

사용자의 할 일을 관리하고, 추가된 할 일에 대해 AI가 다정한 응원과 칭찬 코멘트를 남겨주는 아기자기한 웹 애플리케이션입니다. 

---

## 💡 앱 소개
본 앱은 바쁜 일상 속에서 놓치기 쉬운 할 일들을 직관적으로 기록하고 관리할 수 있도록 돕는 서비스입니다. 은은한 파스텔 톤과 하트/별 패턴의 비주얼을 적용하여 사용자가 편안한 마음으로 하루의 목표를 다듬을 수 있도록 디자인되었습니다. 

* **핵심 기능:** 할 일 추가, 터치 시 완료 표시(💖 토글 및 취소선 효과), 할 일 삭제

---

## 🤖 사용한 AI 및 초기 프롬프트 설정

프로젝트의 몰입감과 재미를 더하기 위해 **Google Gemini AI**를 연동하였습니다. 사용자가 할 일을 입력창에 넣으면 백엔드 서버 없이 브라우저에서 직접 AI API를 호출하여 실시간으로 반응합니다.

### 🤫 처음 입력한 프롬프트 (System Prompt)
AI의 정체성과 답변 규칙을 정의하기 위해 내부에 숨겨둔 초기 프롬프트 설정값은 다음과 같습니다.
> "너는 귀여운 할 일 도우미 AI야. 사용자가 해야 할 일을 입력하면, 그 일에 대해 엄청 다정하고 귀엽게 칭찬이나 한 줄 응원(12자 이내)을 이모티콘과 함께 짧게 답해줘. 질문은 하지 마."

---

## 🛠️ 수정 및 개선 요청 내용
초기 버전의 레이아웃에서 텍스트 시인성을 높이기 위해 다음과 같은 화면 확대 및 UI 최적화 수정을 진행했습니다.

1. **메인 타이틀 확대:** 메인 제목(`h1`)의 크기를 기존 `21px`에서 `32px`로 대폭 확대하여 시선을 사로잡도록 수정
2. **입력창 및 버튼 최적화:** 사용자가 타이핑하기 편하도록 입력 폼과 추가 버튼의 글자 크기를 `18px`로 키우고 패딩(여백) 조절
3. **목록 글자 크기 조정:** 등록된 할 일 텍스트(`.todo-text`)의 기본 크기를 `18px`로 변경하여 가독성 확보
4. **레이아웃 비율 유지:** 글자가 커짐에 따라 전체 카드가 답답해 보이지 않도록 `max-width`를 `440px`로 확장 및 내부 여백 밸런스 조정

## 🧠 어려웠던 점 및 배운 점

### 1. 어려웠던 점
* **포트(Port) 충돌 및 로컬 실행:** 터미널에서 `npx serve`나 파이썬 명령어로 인덱스 파일을 열 때, 간혹 기존에 켜두었던 프로세스가 포트를 점유하고 있어 서버가 열리지 않는 오류를 겪었습니다. 터미널 명령어를 통해 사용 중인 PID를 찾아 강제 종료(`taskkill` / `kill -9`)하는 법을 익히며 해결했습니다.
* **AI 프롬프트 엔지니어링:** AI가 간혹 12자 제한을 넘겨 너무 긴 대답을 하거나 질문을 되받아치는 현상이 있었습니다. 시스템 지침(System Instruction)에 규칙을 명확하고 엄격하게 쥐여주는 프롬프트 작성법의 중요성을 깨달았습니다.

### 2. 배운 점
* **Markdown 활용법:** 프로젝트의 얼굴인 `README.md` 파일을 직접 작성해 보며 `#`, `##` 같은 제목 태그와 인용구(`>`), 코드 블록 표현법 등 마크다운 문법을 완벽히 다룰 수 있게 되었습니다.
* **Vanilla JS 기반의 DOM 조작:** 자바스크립트로 동적으로 `<li>`와 `<span>` 엘리먼트를 생성하고, 클릭 이벤트를 결합하여 완료 상태를 토글하는 웹 프론트엔드의 기초 흐름을 확실하게 이해했습니다.
